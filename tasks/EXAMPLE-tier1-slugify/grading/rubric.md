# Quality rubric (secondary, 0–3) — EXAMPLE task

Applied by the judge in addition to the pass/fail gate. Never overrides `pass`.

- **0** — does not pass hidden tests.
- **1** — passes tests but sloppy (e.g. relies on a heavy regex hack, unreadable, mutates unexpected state).
- **2** — passes, clean and idiomatic, single clear implementation.
- **3** — passes, clean, and handles the rules elegantly (e.g. one well-chosen normalise → replace → trim pipeline) with no dead code.
